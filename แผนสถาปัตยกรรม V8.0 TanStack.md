## **1\. 🌍 The Vision: "Total Type-Safety & Zero Magic"**

ระบบต้องเป็น **Type-Safe** ตั้งแต่ฐานรากจนถึงหน้าจอ (End-to-End) โดยไม่มีช่องว่างให้ Error เล็ดลอดไปได้

* **Absolute Type Integrity:** ทุก Route, Parameter และ API Response ต้องมี Type กำกับ 100%  
* **Edge-Native Performance:** รันบน Cloudflare Pages ด้วยความเร็วสูงสุดผ่าน Nitro Engine  
* **Unified Intelligence:** Content ทั้งหมดถูกส่งจาก Sanity ในรูปแบบ Structured Data ที่พร้อมใช้งานทั้งบน Web และ Mobile ในอนาคต

**4\. 🧠 Data & Routing Protocols**

### **4.1 Type-Safe Routes**

* ทุก Route ต้องใช้ createFileRoute และต้องกำหนด validateSearch ด้วย **Zod Schema** เสมอ  
* การทำ Navigation ห้ามใช้ String Hardcoded ให้ใช้ Link component ที่ระบุ to เป็น Route object เท่านั้น

### **4.2 Data Fetching (The Loader Pattern)**

* ข้อมูลจาก Sanity ต้องถูกดึงผ่าน loader ของ TanStack Router เพื่อให้ข้อมูลพร้อมก่อนที่หน้าจอจะ Render (Zero Layout Shift)  
* ข้อมูลที่ต้องการ Real-time หรือ User Interaction ให้ใช้ **TanStack Query** ร่วมกับ Server Functions

**6\. 🤖 Agent Coding Protocols (The Absolute Rules)**

**กฎเหล็กที่ Agent ต้องตรวจสอบตัวเองอย่างเคร่งครัดก่อนส่งโค้ดทุกครั้ง:**

1. **Edge Compatibility:** ห้ามใช้ Node.js Native APIs (เช่น fs, path, dns) ในฝั่ง Web เพราะต้องรันบน Cloudflare Edge Runtime  
2. **Shared Types Only:** ห้ามประกาศ Type ของข้อมูล (Data Interface) ซ้ำซ้อนในโฟลเดอร์ apps ทุกอย่างที่เกี่ยวกับ Data จาก CMS ต้องอยู่ใน packages/api-client  
3. **Strict shadcn Usage:** การเพิ่ม UI ใหม่บน Web ต้องใช้คำสั่ง npx shadcn@latest add เท่านั้น ห้ามเขียน Copy-paste คอมโพเนนต์มาตรฐานเอง  
4. **Absolute No Any:** ห้ามใช้ any หรือ unknown โดยไม่มีการทำ Type Narrowing เด็ดขาด ทุก Input/Output ต้องผ่าน Zod Validation  
5. **Component Modularization:** แยก Component ย่อยทันทีถ้าส่วน Logic (ก่อนบรรทัด return) ยาวเกิน **50 บรรทัด** หรือไฟล์รวมยาวเกิน **100 บรรทัด**  
6. **Type-Safe Links:** ห้ามใช้ \<a href="..."\> ให้ใช้ \<Link to="..."\> จาก @tanstack/react-router เพื่อรักษาความปลอดภัยของ Type ในการนำทาง  
7. **Zod Search Params:** ทุกหน้าที่รับ Query Params ต้องมี Zod Validation กำกับเสมอ เพื่อป้องกันระบบแครชจาก URL ที่ผิดรูปแบบ  
8. **Streaming Safety:** คอมโพเนนต์ที่ดึงข้อมูลช้า ต้องถูกหุ้มด้วย \<Suspense\> และจัดการ Error Boundary ให้เรียบร้อย
